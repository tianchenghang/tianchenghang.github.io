# Promise, async/await

**executor**

```js
const executor = (resolve, reject) => {};
new Promise(executor);
```

### Promise 对象的两个属性

**PromiseState**: Promise 对象的状态

- pending: Promise 对象初始化时的 PromiseState
- fulfilled: 异步任务成功
- rejected: 异步任务失败

**PromiseResult**: Promise 对象的结果

PromiseResult 即

- resolve(value) 的 value, reject(reason) 的 reason
- onfulfilled, onrejected 回调函数返回 return 或抛出 throw 的值

**resolve 函数** (异步任务成功)

1. 修改 Promise 对象的 PromiseState: `pending ==> fulfilled`
2. 设置 Promise 对象的 PromiseResult = resolve 函数的参数值

**reject 函数** (异步任务失败)

1. 修改 Promise 对象的PromiseState: `pending ==> rejected`
2. 设置 Promise 对象的PromiseResult = reject 函数的参数值

### node 的 promisify

将普通函数转换为 Promise 风格的函数

```js
const util = require("util");
const _readFile = util.promisify(fs.readFile);
_readFile("./README.md").then(
  (value) => {
    console.log(value.toString().slice(0, 10));
  },
  (reason) => {
    console.log(reason);
  },
);
```

### PromiseState: Promise 对象的状态

**PromiseState 的切换**

- 从 pending 到 fulfilled 成功
- 从 pending 到 rejected 失败
- 一个 Promise 对象的 PromiseState 只能切换一次

**onfulfilled, onrejected 回调函数**

1. 如果回调函数 (表面) 抛出一个对象 reason, 则 (实际) 返回的 promise 的 PromiseState 为 rejected, PromiseResult 为 reason
2. 如果回调函数 (表面) 返回一个非 Promise 对象 value, 则 (实际) 返回的 promise 的 PromiseState 为 fulfilled, PromiseResult 为 value
3. 如果回调函数 (表面) 返回 Promise 对象, 则 (实际) 返回该 Promise 对象
4. 如果回调函数中 throw what, 则 then, catch 返回 Promise{ PromiseState: rejected, PromiseReturn: what }
5. 如果回调函数中 return what (没有 return 等价于 return undefined), 则 then, catch 返回 Promise{ PromiseState: fulfilled, PromiseReturn: what }

### Promise 的方法

1. Promise.resolve(value)
   - 无参数, 或传递一个非 Promise 对象, 返回的 promise 的 PromiseState 为 fulfilled, PromiseReturn 为 value
   - 传递一个 Promise 对象, 返回该 Promise 对象
2. Promise.reject(reason) 返回的 promise 的 PromiseState **始终**为 rejected, PromiseReturn 为 reason
3. Promise.catch(onrejected) 指定异步任务失败时, 即 `reject(?); throw ?` 时的错误处理函数
4. Promise.finally(onFinally) 返回的 promise 的 PromiseState 始终为 fulfilled

**等价关系**

- Promise.resolve(value) 等价于 new Promise((resolve, reject) => resolve(value))
- Promise.reject(reason) 等价于 new Promise((resolve, reject) => reject(reason))
- reject(reason) 等价于 throw reason
- promise.catch(onrejected) 等价于 promise.then(undefined, onrejected)

`Promise.all(), Promise.any(), Promise.race(), Promise.allSettled`: 将一组子 promise 包装为一个新的 promise (将一组子异步任务包装为一个新的异步任务)

1. Promise.all()
   - 有一个子 promise 失败时, 返回的 promise 失败 rejected
   - 全部子 promise 都成功时, 返回的 promise 才成功 fulfilled
2. Promise.any()
   - 有一个子 promise 成功时, 返回的 promise 成功 fulfilled
   - 全部子 promise 都失败时, 返回的 promise 才失败 rejected
3. Promise.race()
   - 最先切换 PromiseState 的子 promise 成功时, 返回的 promise 成功 fulfilled
   - 最先切换 PromiseState 的子 promise 失败时, 返回的 promise 失败 rejected
4. Promise.allSettled()
   - settled: 包含 fulfilled 和 rejected 两种情况
   - 全部子 promise 是否都 "已解决" (可能成功, 可能失败)
   - 返回的 promise 始终成功 fulfilled

### 终止 Promise 链

有且只有一种方式: 返回一个 PromiseState = pending 的 promise 对象, 且 finally 块的无参回调函数**不会**被执行

### async/await

async/await 是 ES7 提出的

- async 函数始终返回一个 Promise 对象, await 只能用在 async 函数体中
- 如果 async 函数 (表面) 返回一个非 Promise 对象 value, 则 (实际) 返回 `Promise{ PromiseState: 'fulfilled',
PromiseResult: value }`
- 如果 async 函数 (表面) 返回一个 Promise 对象, 则 (实际) 返回该 Promise 对象
- 如果 async 函数 (表面) 抛出一个对象 reason, 则 (实际) 返回 `Promise{ PromiseState: 'rejected',
PromiseResult: reason }`
- await 一个返回 fulfilled 的 Promise 对象的 async 函数时, 返回 promise.PromiseReturn
- await 一个返回 rejected 的 Promise 对象的 async 函数时, 抛出错误

```js
async function f() {
  /* 表面上 */ return new Error("what");
  /* 实际上 */
  // return Promise{ PromiseState: 'fulfilled', PromiseResult: Error{'what'} }
}
```

对比代码

```ts
type Callback = (err: Error, ...args: any[]) => any;
type Thunk = (cb: Callback) => any;
type Thunkify = (fn: (...args: any[]) => any) => (...args: any[]) => Thunk;
```

```js
const fs = require("fs");
const thunkify = require("./snippet/thunkify");
let readFileThunk = thunkify(fs.readFile);
function* genFunc() {
  let data1 /* Buffer */ = yield readFileThunk("./package.json");
  console.log(data1.toString());
  let data2 /* Buffer */ = yield readFileThunk("./README.md");
  console.log(data2.toString());
}
// 参考 ch18 Generator, thunkify
```

```js
const fs = require("fs").promises;
async function _readFile() {
  let data1 = await fs.readFile("./package.json");
  console.log(data1.toString());
  let data2 = await fs.readFile("./README.md");
}
_readFile();
// 参考 ch18 Generator, thunkify
```

Generator 对比 async

1. 语义化: function\* => async function, yield => await
2. 生成器函数需要执行器 (参考 ch18 Generator, thunkify), async 函数不需要
3. async 函数始终返回一个 Promise 对象

```js
function timeout(ms) {
  return new Promise((resolve, reject) => {
    setTimeout(resolve, ms);
  });
}
async function logAsync(val, ms) {
  await timeout(ms);
  console.log(val);
}
logAsync("hello world", 3000);
```

等价于

```js
async function timeout(ms) {
  await new Promise((resolve, reject) => {
    setTimeout(resolve, ms);
  });
}
async function logAsync(val, ms) {
  await timeout(ms);
  console.log(val);
}
logAsync("hello world", 3000);
```
